# Codex 自动化：每天 00:00 生成 GPT 日记回应

## 推荐运行时间

```cron
0 0 * * *
```

也就是每天 00:00 运行。

## Automation Prompt

```text
每天 00:00 运行。

使用 $diary-response 处理昨天的 Obsidian 日记。
如果没有明确指定路径，请根据当前日期计算昨天，目标路径格式为：
01日记/YYYY-MM-DD.md

完成后直接写回文件。
```

## 为什么是“昨天”

00:00 的时候，新的一天刚开始，但刚结束、内容完整的是昨天。

例如：

- 2026-06-09 00:00 运行；
- 处理文件：`01日记/2026-06-08.md`。

Skill 里也写了这个规则：自动化运行且没有指定路径/日期时，处理昨天的日记。

## 自动化做什么

1. 找到昨天的日记文件。
2. 读取 `## 碎念流`。
3. 如有本地图片且工具可以读取，就把图片也当成当天材料。
4. 可以少量在 `## 碎念流` 原文上添加 Hover Reveal 批注。
5. 在 `## 长文` 下写 GPT 的整体回应。
6. 替换原有 `## 长文` 内容。

## 注意

- 自动化 prompt 不要写得太复杂。真正的写作规则放在 `$diary-response` skill 里。
- 如果 prompt 和 skill 同时写很多风格要求，模型容易开始“表演规则”，反而写得更假。
